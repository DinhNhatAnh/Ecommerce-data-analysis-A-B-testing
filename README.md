# E-commerce Data Analysis & A/B Testing

## 📌 1. Project Overview
This project analyzes e-commerce user behavior and evaluates a new website design using A/B testing. The goal is to determine whether the new version improves conversion rates and to provide actionable business recommendations.
## 🎯 2. Business Problem
The company launched a new website interface (Version B) and wants to determine whether it performs better than the current version (Version A) in terms of conversion rate.
## 📊 3. Dataset
- Simulated A/B testing dataset (10,000 users)
- Features:
  - user_id
  - group (A/B)
  - device (mobile/desktop)
  - day
  - converted (0/1)
## 🛠 4. Tools & Technologies
- SQL: Data extraction & aggregation
- Python: Data analysis & statistical testing
- Excel: Quick validation & pivot tables
- Power BI: Dashboard visualization
## 🔄 5. Workflow
# 5.1 Data Cleaning
- Removed duplicates
- Filtered valid A/B groups
# 5.2 SQL Analysis
- Conversion rate by group
- User distribution
# 5.3 Python Analysis
- A/B testing (Z-test)
- Confidence interval
- Effect size
- Power analysis
# 5.4 Visualization
- Conversion rate comparison
- Segmentation by device
- Daily trend
## 📈 6. Key Insights
- Version B shows a higher conversion rate than Version A
- Conversion improvement is statistically significant (p < 0.05)
- Mobile users behave differently than desktop users
- Conversion trends remain consistent over time
## 💡 7. Business Recommendations
- Roll out Version B to all users
- Optimize mobile experience further
- Continue A/B testing for additional features
